## Docker 部署

### 一鍵部署

在任何安裝了 Docker 的機器上執行：

```bash
curl -fsSL https://raw.githubusercontent.com/jheweilin/xlab/main/scripts/docker-deploy.sh | bash
```

### 手動部署

```bash
# 1. 克隆專案
git clone https://github.com/jheweilin/xlab.git
cd xlab

# 2. 建置 Docker 映像
./scripts/docker-build.sh

# 3. 啟動服務
./scripts/docker-start.sh
```

### Docker 管理指令

| 指令 | 說明 |
|------|------|
| `./scripts/docker-start.sh` | 啟動服務 |
| `./scripts/docker-stop.sh` | 停止服務 |
| `./scripts/docker-restart.sh` | 重啟服務 |
| `./scripts/docker-build.sh` | 重新建置映像 |
| `docker compose logs -f` | 查看即時日誌 |

### Docker 資料持久化

資料儲存在 `data/` 目錄：

```
data/
├── uploads/    # 上傳的圖片
└── db/         # SQLite 資料庫
```

### 自訂端口

```bash
XLAB_PORT=8080 ./scripts/docker-start.sh
```

## 服務管理

### 腳本位置

所有管理腳本位於 `scripts/` 目錄：

```
scripts/
├── start.sh    # 啟動服務
├── stop.sh     # 停止服務
├── restart.sh  # 重啟服務
├── status.sh   # 查看狀態
├── logs.sh     # 查看日誌
└── setup.sh    # 初始化設定
```

### 啟動服務

```bash
# 開發模式（預設）
./scripts/start.sh

# 或指定模式
./scripts/start.sh dev

# 生產模式
./scripts/start.sh prod
```

**開發模式 vs 生產模式：**

| 特性 | 開發模式 | 生產模式 |
|------|----------|----------|
| 熱重載 | ✅ 支援 | ❌ 不支援 |
| 效能 | 較慢 | 較快 |
| 錯誤顯示 | 詳細 | 簡化 |
| 適用場景 | 開發測試 | 正式上線 |

### 停止服務

```bash
./scripts/stop.sh
```

### 重啟服務

```bash
# 開發模式重啟
./scripts/restart.sh

# 生產模式重啟
./scripts/restart.sh prod
```

### 查看狀態

```bash
./scripts/status.sh
```

輸出包含：
- 服務運行狀態
- PID 資訊
- 端口佔用情況
- HTTP 回應檢查
- 最近日誌

### 查看日誌

```bash
# 顯示最後 50 行
./scripts/logs.sh

# 顯示最後 100 行
./scripts/logs.sh 100

# 持續監控日誌
./scripts/logs.sh -f

# 組合使用
./scripts/logs.sh -n 200 -f
```

## 設定說明

### 環境變數

可透過環境變數自訂配置：

| 變數名稱 | 預設值 | 說明 |
|----------|--------|------|
| XLAB_PORT | 4567 | 服務端口 |
| XLAB_HOST | 0.0.0.0 | 監聽位址 |

使用範例：

```bash

```

```

├── src/                    # 源碼目錄
│   ├── app/                # Next.js App Router
│   │   ├── (frontend)/     # 前台頁面
│   │   ├── (admin)/        # 後台頁面
│   │   └── api/            # API 路由
```

1. **儀表板** - 系統概覽

### 新增商品流程

1. 登入後台 → 商品管理 → 新增商品
2. 填寫商品資訊：
   - 名稱（必填）
   - 分類（必填）
   - 描述
   - 詳細內容
   - 價格
   - 標籤
3. 上傳商品圖片
4. 設定是否精選/上架
5. 儲存

### 管理分類

1. 登入後台 → 分類管理
2. 可建立階層式分類（父分類/子分類）
3. 設定分類圖片和說明

## 常見問題

### Q1: 服務無法啟動，顯示端口被佔用

```bash
# 檢查佔用端口的程序
lsof -i :4567

# 強制停止
./scripts/stop.sh

# 或手動終止
kill -9 <PID>
```

### Q2: 網頁無法訪問

1. 確認服務是否運行：
   ```bash
   ./scripts/status.sh
   ```

2. 檢查防火牆設定：
   ```bash
   # macOS
   sudo pfctl -sr | grep 4567
   ```

3. 確認監聽位址：
   - `0.0.0.0` = 允許外部訪問
   - `127.0.0.1` = 僅本機訪問

### Q3: 資料庫錯誤

```bash
# 重新生成 Prisma Client
npx prisma generate

# 重新同步資料庫
npx prisma db push

# 重新建立種子資料
npx tsx prisma/seed-xlab.ts
```

### Q4: 圖片無法顯示

1. 確認圖片目錄存在：
   ```bash
   ls -la public/uploads/
   ```

2. 檢查檔案權限：
   ```bash
   chmod -R 755 public/uploads/
   ```

### Q5: 如何備份資料？

```bash
# 備份資料庫
cp prisma/dev.db prisma/dev.db.backup

# 備份上傳檔案
cp -r public/uploads/ uploads-backup/
```

### Q6: 如何還原資料？

```bash
# 停止服務
./scripts/stop.sh

# 還原資料庫
cp prisma/dev.db.backup prisma/dev.db

# 還原上傳檔案
cp -r uploads-backup/* public/uploads/

# 重啟服務
./scripts/start.sh
```

## 維護指南

### 定期維護項目

| 項目 | 頻率 | 指令 |
|------|------|------|
| 檢查服務狀態 | 每日 | `./scripts/status.sh` |
| 查看錯誤日誌 | 每日 | `./scripts/logs.sh \| grep -i error` |
| 備份資料庫 | 每週 | `cp prisma/dev.db backups/` |
| 清理舊日誌 | 每月 | `> logs/xlab.log` |
| 更新依賴 | 每月 | `npm update` |

### 日誌管理

```bash
# 查看日誌大小
du -sh logs/xlab.log

# 清空日誌（保留檔案）
> logs/xlab.log

# 備份並清空日誌
mv logs/xlab.log logs/xlab.log.$(date +%Y%m%d)
touch logs/xlab.log
```

### 效能監控

```bash
# 查看服務記憶體使用
ps aux | grep node

# 查看端口連線數
netstat -an | grep 4567 | wc -l
```

### 系統更新

```bash
# 1. 停止服務
./scripts/stop.sh

# 2. 備份資料
cp prisma/dev.db prisma/dev.db.backup

# 3. 更新代碼（如使用 Git）
git pull

# 4. 更新依賴
npm install

# 5. 更新資料庫（如有變更）
npx prisma db push

# 6. 重啟服務
./scripts/start.sh prod
```
